Structural patterns that make this work
1. Use “hub pages” instead of deep menus

Each top-level item should land on a hub page that:

summarizes the section
links clearly to subpages
highlights key actions

Example:
“Get Help” page shows:

big “Report Incident” button
short explanation
links to FAQ + resources
2. Prioritize user intent (not org structure)

Users typically come to:

Report something
Find events
Get involved
Understand the commission

Your nav reflects that—not internal committees.

3. Keep dropdowns shallow

Avoid more than:

5–6 items per dropdown
1 level deep (no nested menus)
4. Add cross-links instead of duplicating nav

Example:

Events page links to “Volunteer”
Programs page links to “Join a Working Group”
Report page links to “Resources”

This reduces menu clutter while preserving discoverability.

5. Use a persistent call-to-action

Add a visually distinct button in the header:

“Report an Incident”

This removes pressure from the nav to carry that weight.


Recommended structure (balanced)
Top-level pages:
Home
About (single page)
Get Help (separate pages for reporting flow)
Programs (start as single page)
Events (separate system)
Get Involved (single page)
UX patterns to use inside pages
1. Anchor navigation (very effective)

At top of long pages:

Mission
Members
History
Structure

Click scrolls to section.

2. Expandable sections (accordion)

Use for:

FAQs
Dense bios
Process explanations

Avoid overusing—too many hidden sections reduces discoverability.

3. Card-based sections

On hub pages (like Programs):

Each program = a card
Click → expands or opens detail page
Decision rule (simple)

Use this test:

Make it a separate page if:
A user might bookmark or share it
It supports a single clear task
It may grow significantly
Keep it on a shared page if:
It’s contextual or background
It’s short
Users don’t need it often
Common mistake to avoid

Don’t mirror internal structure (e.g., each working group = a page) unless:

each has substantial, updated content

Otherwise, it creates:

thin pages
harder navigation
maintenance overhead
Bottom line
About + Get Involved → single pages with sections
Help + Events → separate, task-focused pages
Programs → start as one page, expand later if needed
